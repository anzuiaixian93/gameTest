# 效果图需求：百分百空手接白刃

本文给美术朋友使用。当前 Cocos 原型只是技术占位，不代表最终画面。后续代码实现必须以这里提交的效果图为准，不要让 Codex 自行脑补最终效果。

## 目标

先把核心玩法画清楚：玩家在手机竖屏中观察武士下劈，判断真刀落点，点击屏幕让侠客猫空手接白刃。

效果图不需要一开始就精修，但必须能让程序理解：角色位置、刀的运动路径、点击时机、成功/失败反馈、按钮和提示信息放在哪里。

## 画布规格

- 方向：手机竖屏
- 建议尺寸：750 x 1334
- 安全区域：顶部和底部留出微信小游戏常见遮挡空间，不要把关键按钮贴边
- 文件格式：PNG 或 JPG
- 草图、线稿、灰盒图都可以，先以表达清楚为主

## 第一批必须提供的效果图

### 1. 主游戏画面

用途：决定游戏第一眼长什么样。

需要标清楚：

- 侠客猫站位
- 武士站位
- 刀的初始位置
- 最佳接刀区域或提示线
- 当前关卡显示位置
- 提示文字位置
- 是否需要开始按钮、重试按钮、下一关按钮
- 背景大概氛围

建议文件名：`concept_main_game_001.png`

### 2. 出刀前准备画面

用途：决定倒计时、准备动作和紧张感。

需要标清楚：

- 猫的准备姿势
- 武士举刀姿势
- 倒计时或提示文字怎么显示
- 画面是否需要轻微抖动、聚焦、暗角

建议文件名：`concept_ready_001.png`

### 3. 刀下落过程图

用途：决定核心点击判定和动画路径。

至少画 3 个关键帧：

- 刀刚开始下落
- 刀落到中途
- 刀接近最佳点击点

需要标清楚：

- 刀从哪里到哪里
- 真刀白光出现在哪一段
- 假动作和真动作视觉上怎么区分
- 最佳点击点在哪

建议文件名：`concept_blade_motion_001.png`

### 4. 成功接住画面

用途：决定玩家爽感。

需要标清楚：

- 猫的成功动作
- 刀停在哪里
- 火花、白光、冲击线的位置
- 成功文字或通关提示
- 下一关按钮位置

建议文件名：`concept_success_001.png`

### 5. 失败画面

用途：决定失败反馈是否好笑、是否清楚。

至少画一种失败，最好画两种：

- 点早：猫抓空、武士偷笑、提示“早了”
- 点晚：猫被拍飞或震到、提示“晚了”

需要避免血腥，走夸张喜剧表达。

建议文件名：

- `concept_fail_early_001.png`
- `concept_fail_late_001.png`

### 6. 结算和重试界面

用途：决定一局结束后的操作流程。

需要标清楚：

- 成功后显示什么
- 失败后显示什么
- 重试按钮位置
- 下一关按钮位置
- 是否有分享按钮
- 是否显示误差毫秒，例如“早了 120ms”

建议文件名：`concept_result_panel_001.png`

### 7. UI 元素拆分图

用途：让程序按统一风格做按钮、弹窗、关卡牌。

需要画出：

- 开始按钮
- 重试按钮
- 下一关按钮
- 关卡牌
- 成功提示框
- 失败提示框
- 音效开关
- 震动开关

建议文件名：`concept_ui_elements_001.png`

## 标注要求

每张效果图请直接在图上写标注，程序更容易看懂。

推荐标注：

- “猫在这里”
- “武士在这里”
- “刀从这里落下”
- “黄线是最佳点击区”
- “这里出现白光”
- “这里屏幕震动”
- “失败后按钮在这里”
- “这个是假动作，不要出现白光”

如果某个元素只是临时示意，也请写“临时”。
